## О проекте (коротко)

Beautiful Grid — open-source десктоп-приложение для фотографов: импорт сотен фото → сетка → автосортировка по цвету → экспорт. Стек: **Tauri (Rust) + React 18 + TypeScript + Vite + Tailwind + Zustand**. Полностью офлайн, MIT, Mac (arm64) + Windows. Полная спецификация — [docs/PRD.md](docs/PRD.md).

## Правила работы

1. **Никогда не пушить в `main` напрямую.** Любая работа — отдельная ветка + Pull Request. Автор мержит сам.
2. **После каждой релизной версии** (v1.0.0, v1.1.0, ...):
   - добавить запись в `CHANGELOG.md` (формат [Keep a Changelog](https://keepachangelog.com/));
   - сменить статус спринта в [docs/Roadmap.md](docs/Roadmap.md) с `Planned` → `Done`;
   - поставить git-тег `vX.Y.Z` → CI собирает DMG/MSI и публикует в GitHub Releases.
3. **Не нарушать инварианты PRD §9:**
   - Никаких сетевых запросов с пользовательскими данными — приложение полностью офлайн.
   - Оригинальные фото никогда не модифицируются и не удаляются (только чтение + копирование при экспорте).
4. **Не плодить документы в `docs/`** без согласования. Перед созданием нового `.md` — проверь, нельзя ли расширить существующий.
5. **Любой новый `docs/*.md`** должен начинаться с шапки:
   ```markdown
   # Название документа
   > **О документе:** 2-3 строки о содержимом.
   > **Связанные документы:** [Doc1.md](Doc1.md) · [Doc2.md](Doc2.md)
   ```
6. **Идеи и проблемы — в [docs/Ideas.md](docs/Ideas.md)**, не в PRD. PRD обновляется только при значимых продуктовых изменениях.

## Модульность кода

### Лимиты файла

Разные планки для разных стеков — у Rust естественно больше «обвязки» (`use`, derive-блоки, реализации `Display`/`Error`, inline-тесты в `#[cfg(test)] mod tests`).

| Стек | Цель | Жёсткий лимит | Комментарий |
|---|---|---|---|
| **Rust (`src-tauri/`)** | 400 строк | **600 строк** | Тесты в том же файле НЕ считаем — если основной модуль 350 + тесты 200, это норм. |
| **Frontend (`src/`, TS/TSX)** | 250 строк | **400 строк** | Компоненты должны быть мелкими; превышение почти всегда значит, что пора выделить хук/подкомпонент. |
| **Конфиги (`*.json`, `*.toml`)** | — | — | Лимиты не применяются. |

**Правила:**
- Одна ответственность на файл.
- В начале файла — короткий комментарий (`/** ... */` для TS, `//! ...` для Rust) о назначении модуля.
- Превышение жёсткого лимита → split. Если split невозможен — оставить комментарий с обоснованием почему.

### Структура frontend (`src/`)
- `components/` — переиспользуемые React-компоненты.
- `pages/` — Library / Grid / Export.
- `store/` — Zustand-сторы.
- `hooks/` — кастомные хуки.
- `lib/` — utils, чистые функции.
- `types/` — общие TypeScript-типы.

### Структура backend (`src-tauri/src/`)
- `commands/` — Tauri-команды (`import_photos`, `analyze_colors`, `export_grid`, ...).
- `image/` — thumbnails, color extraction.
- `project/` — `.bgrid` сериализация.
- `main.rs` — только точка входа (`fn main`, регистрация команд).

### Запрещено
- Циклические импорты (TS и Rust).
- `export *` / `pub use ... *` без явной причины (barrel-файлы для публичного API модуля — ок).
- Мутируемые глобалы. Только `const` / `static`.
- **Хардкод URL платформ донатов** (Buy Me a Coffee / Boosty / GitHub Sponsors) — выносим в `src/config/` или `tauri.conf.json`, чтобы менять без правки логики.
- Прямое чтение/запись оригинальных фото пользователя из любого места, кроме модуля `image/` (страховка от случайного нарушения инварианта).
