## Sprint 0 — Setup проекта `[v1.0 MVP]` · `Done` (v0.1.0-alpha)

**Цель.** Подготовить фундамент проекта: рабочий каркас Tauri+React, CI, лицензия, README, иконка. После этого спринта можно писать продуктовые фичи.

**Definition of Done.**
- Репозиторий `beautiful-grid` создан на GitHub под лицензией MIT.
- `npm run tauri dev` запускает пустое окно приложения на macOS (arm64) и Windows.
- CI на GitHub Actions собирает обе платформы при пуше в `main`.
- README содержит описание, скриншот, инструкцию запуска и контрибьюшна.
- Финальная иконка приложения утверждена (или зафиксирован placeholder + задача на замену).

**Задачи.**

*Frontend:*
- [x] Инициализировать React 18 + TypeScript + Vite.
- [x] Подключить Tailwind CSS, базовая тёмная тема.
- [x] Подключить Zustand, пустой store.
- [x] Каркас навигации: Library / Grid / Export — пустые экраны.

*Rust backend (Tauri):*
- [x] Инициализировать `src-tauri/`, проверить сборку под arm64 и x64.
- [x] Скелет команд (`commands/mod.rs`) — пустые stubs `import_photos`, `analyze_colors`, `export_grid`.
- [x] `tauri.conf.json`: заголовок окна, размер, иконки.

*Infra + CI:*
- [x] GitHub Actions: сборка под macOS (arm64) и Windows на каждый PR.
- [x] Линтеры: ESLint + Prettier (frontend), rustfmt + clippy (backend).
- [x] Файл `LICENSE` (MIT).

*Docs:*
- [x] README: что это, скриншот, как запустить.
- [x] CONTRIBUTING.md: code style, ветки, PR-процесс.

## Sprint 1 — MVP core `[v1.0 MVP]` · `Planned`

**Цель.** Реализовать ключевой пользовательский поток PRD §5.1: импорт фото → библиотека → грид → автосортировка по цвету → drag&drop → экспорт одной картинкой → сохранение проекта.

**Definition of Done.**
- На 300 JPG/PNG: импорт, миниатюры и доминирующий цвет генерируются с прогрессом формата `25 / 600`.
- В сетке N в ряд (1–10) drag&drop без видимых лагов на 500 фото (виртуализация списка).
- Кнопка «Сортировать по цвету» переставляет фото по HSL-градиенту.
- Экспорт сетки 4×N (N=20–30) в PNG/JPG < 15 сек.
- Проект сохраняется в `.bgrid` и открывается обратно с тем же гридом.
- Кнопка «Поддержать» во всех трёх местах PRD §7 + одноразовое приветственное модальное окно при первом запуске.
- DMG + MSI публикуются в GitHub Releases как **v1.0.0**.

**Задачи.**

*Frontend:*
- [ ] Library: drag&drop папки/файлов, виртуализированный grid превью.
- [ ] Grid: настройка N в ряд, drag&drop через `dnd-kit`.
- [ ] Export: предпросмотр, выбор формата (PNG/JPG), качество JPG.
- [ ] Прогресс импорта/анализа (общий процент + счётчик).
- [ ] Кнопка «Поддержать» (header / About / Settings) → выбор платформы во внешнем браузере.
- [ ] Приветственное модальное окно при первом запуске + флаг в локальных настройках.
- [ ] Сохранение / открытие проекта (`.bgrid`).

*Rust backend:*
- [ ] Команда `import_photos`: чтение, генерация миниатюр в `.cache/` (WebP).
- [ ] Команда `analyze_colors`: доминирующий цвет (`kmeans_colors` или `color-thief-rs`), параллелизация через `rayon`.
- [ ] Команда `export_grid`: склейка в одно изображение через `image` crate, тайловая запись.
- [ ] Сериализация `.bgrid` (serde).
- [ ] Сортировка по HSL-градиенту + тесты.

*Infra + CI:*
- [ ] GitHub Actions: автоматическая сборка релизных артефактов при теге `v*`.
- [ ] Подпись DMG (Apple notarization при наличии сертификата, иначе — документируем "self-signed").
- [ ] Установщик MSI с базовой конфигурацией.

*Docs:*
- [ ] README с GIF-демо основного потока.
- [ ] CHANGELOG.md: первая запись v1.0.0.
- [ ] Видео-туториал 2–3 минуты (YouTube) с основным потоком.

## Sprint 2 — Groups + rename export `[v1.1 Post-MVP]` · `Planned`

**Цель.** Дать пользователю организовать большую подборку по группам (Сборы / Церемония / Прогулка) и экспортировать переименованные копии файлов по шаблону.

**Definition of Done.**
- Можно создать / переименовать / удалить группу.
- Каждое фото принадлежит **строго одной группе** (или null).
- Экспорт «переименовать»: выбор целевой папки, шаблон с `{group}`, `{index:003}`, `{date}`, `{original}`, предпросмотр, копирование.
- Оригиналы не модифицируются (тест: checksum до и после).

**Задачи.**

*Frontend:*
- [ ] Боковая панель групп (CRUD).
- [ ] Drag фото между группами.
- [ ] Экспорт-модал: builder шаблона имени, предпросмотр первых 10 имён.

*Rust backend:*
- [ ] Команда `export_renamed`: создание папки, копирование, генерация имён по шаблону.
- [ ] Шаблонизатор имён (`{var}` → значение).
- [ ] Парсинг EXIF-даты (`kamadak-exif` или аналог).
- [ ] Расширение `.bgrid`: массив `groups`, `group_id` в фото.

*Docs:*
- [ ] CHANGELOG: v1.1.0.
- [ ] README/блог-пост: «новая фича — экспорт с переименованием».

## Sprint 3 — Вариации внутри ряда `[v1.2 Post-MVP]` · `Planned`

**Цель.** Разрешить смешанные ряды (например, 3 вертикальных + 1 горизонтальное при N=4) для более живой композиции.

**Definition of Done.**
- В настройках ряда можно задать паттерн `VVVH` для каждой строки.
- Drag&drop работает корректно с разной шириной ячеек.
- Экспорт корректно собирает смешанные ряды.

**Задачи.**

*Frontend:*
- [ ] UI редактора паттерна ряда (`V`/`H` toggle).
- [ ] Адаптация виртуализации под переменную ширину.

*Rust backend:*
- [ ] Расширение алгоритма склейки на смешанные ряды.
- [ ] Расширение `.bgrid` для хранения паттернов рядов.

*Docs:*
- [ ] CHANGELOG: v1.2.0.

## Sprint 4 — Tags & collections `[v1.3 Post-MVP]` · `Planned`

**Цель.** Дать ручные метки и быстрые фильтры внутри библиотеки проекта.

**Definition of Done.**
- На фото можно повесить N тегов (свободный текст).
- В Library работает фильтр «показать только с тегом X».
- Коллекция = именованный набор фото, отдельная от групп; фото может быть в нескольких коллекциях.

**Задачи.**

*Frontend:*
- [ ] Tag input на превью.
- [ ] Панель фильтров в Library.
- [ ] CRUD коллекций.

*Rust backend:*
- [ ] Расширение `.bgrid`: `tags: string[]`, `collections: { id, name, photo_ids }`.

*Docs:*
- [ ] CHANGELOG: v1.3.0.

## Sprint 5 — Instagram preview `[v1.4 Post-MVP]` · `Planned`

**Цель.** Симуляция вида профиля Instagram 3×N для контент-мейкеров.

**Definition of Done.**
- Отдельный режим «IG preview»: грид жёстко 3 колонки, квадратные ячейки.
- Можно переключаться между «обычным» и «IG» гридом без потери порядка.
- Экспорт IG-сетки в PNG (для презентации клиенту).

**Задачи.**

*Frontend:*
- [ ] Переключатель режима в Grid.
- [ ] Стилизация под IG (квадраты, gap).

*Rust backend:*
- [ ] Опция «square crop» в `export_grid`.

*Docs:*
- [ ] CHANGELOG: v1.4.0.

## Future — открытый бэклог `[Future]`

После Sprint 5 роадмап ведётся через **проблемы пользователей** и **идеи**. Источник — [Ideas.md](Ideas.md).

Кандидаты (из PRD §10, Market Research §10 и PRD §12):
- Поддержка RAW (Canon CR2/CR3, Nikon NEF, Sony ARW).
- Группировка по палитре (тёплые / холодные / монохром / яркие).
- Локальные ML-модели для умной автокомпоновки.
- Расширенные шаблоны раскладки (mood-board, polaroid, hexagon).
- Пресеты соотношений сторон холста для экспорта.
- Локализация (EN/RU как минимум; community-перевод).

Идея → triage в [Ideas.md](Ideas.md) → попадает в новый спринт или в Rejected.

## Как этот документ растёт

1. Любая новая идея сначала падает в [Ideas.md](Ideas.md) → раздел Inbox.
2. Раз в N дней — триаж: переносим в `Triaged` с пометкой «в Sprint X» / «Future» / «Rejected + причина».
3. Когда формируется новый спринт после v1.0 — выбираем 3–5 идей из Triaged + актуальные проблемы пользователей, описываем здесь по шаблону (Цель / DoD / Задачи).
4. PRD обновляем только при значимых продуктовых изменениях, а не при каждой идее.
