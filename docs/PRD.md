## 1. Обзор продукта

**Beautiful Grid** — десктопное open-source приложение для фотографов и дизайнеров, которое помогает компоновать большие подборки фотографий в красивые визуальные сетки. Приложение работает полностью локально (без отправки данных в облако), поддерживает Mac и Windows, и распространяется с открытым исходным кодом на GitHub.

## 2. Проблема и цель

**Проблема.** Фотографы и дизайнеры регулярно делают подборки из сотен фотографий — для портфолио, Instagram-лент, мудбордов, фотокниг. Существующие инструменты либо громоздкие (Photoshop, InDesign), либо ограниченные (онлайн-коллажмейкеры), либо отправляют файлы в облако (что неприемлемо для коммерческой съёмки).

**Цель.** Дать удобный локальный инструмент, который:
- быстро визуально раскладывает большие подборки фото в адаптивную сетку,
- помогает подобрать порядок по цвету/настроению,
- экспортирует результат как в виде целостного изображения, так и в виде набора переименованных файлов в правильном порядке.

## 3. Целевая аудитория

- **Фотографы** (свадебные, портретные, лайфстайл, репортажные) — отбор и оформление фотосессий, подготовка к публикации, генерация портфолио.
- **Дизайнеры** — мудборды, референс-доски, визуальные подборки для клиентов.
- **Контент-мейкеры** — кто планирует ленту в социальных сетях.

## 4. Ключевые функции

### 4.1 MVP (v1.0)
1. **Импорт фотографий** — drag&drop папки/файлов, поддержка JPG и PNG.
2. **Библиотека проекта** — превью всех фото с быстрой навигацией.
3. **Сетка с настройкой количества фото в ряду** — без ограничения по высоте.
4. **Ручной drag & drop** — перетаскивание фото в сетке.
5. **Авто-сортировка по доминирующему цвету** — алгоритм извлекает основной цвет каждого фото и сортирует по похожести (HSL-градиент).
6. **Экспорт результата в одну склеенную картинку** (PNG/JPG).
7. **Сохранение проекта** — возврат к редактированию позже.

### 4.2 Следующие спринты (v1.1+)
- **Группы внутри проекта** — пользователь создаёт именованные группы (например: «Сборы», «Церемония», «Прогулка»), каждое фото принадлежит **строго одной группе**.
- **Экспорт с переименованием** — генерация копии всех фото в новую папку с переименованием по шаблону (см. §5.2). Оригиналы никогда не изменяются.
- **Вариации внутри ряда** — например, при 4 фото в ряд можно поставить 3 вертикальных + 1 горизонтальную.
- **Ручные теги и коллекции** — пометки и фильтры для библиотеки.
- **Группировка по палитре** — кластеризация фото по типу палитры (тёплые/холодные, монохром, яркие).
- **Превью Instagram-ленты** — симуляция вида профиля 3×N.

### 4.3 Поддержка проекта (поперечная фича)
- **Кнопка «Поддержать»** в UI приложения, ведущая на выбор из 3 платформ донатов:
  - **Buy Me a Coffee** — международная разовая поддержка.
  - **Boosty** — для русскоязычной аудитории.
  - **GitHub Sponsors** — для open-source-сообщества.

  Конкретные URL подключаются после релиза v1.0 (см. §12).
- **Размещение в UI:** доступна во всех экранах (например, в шапке окна, в меню «About», в настройках) — пользователь не должен искать кнопку.
- **Одноразовое приветственное окно при первом запуске:** при первой инициализации показывается ненавязчивое модальное окно «Это open-source проект, его можно поддержать» с тремя кнопками платформ и опцией «Закрыть и больше не показывать» (галочка по умолчанию выставлена в «не показывать»). Запоминается флагом в локальных настройках.

## 5. Пользовательские сценарии

### 5.1 Базовый поток (MVP)
1. Пользователь открывает приложение и создаёт новый проект.
2. Перетаскивает в окно папку с 100–500 фотографиями (JPG/PNG).
3. Приложение генерирует миниатюры и извлекает доминирующий цвет каждого фото (фоновая обработка с прогрессом).
4. Пользователь видит сетку, в которой задаёт количество фото в ряду (например, 4).
5. Нажимает «Сортировать по цвету» — приложение перераскладывает фото по похожести цвета.
6. Корректирует порядок вручную через drag & drop.
7. Жмёт «Экспорт» → сохраняет одно большое изображение.
8. Сохраняет проект и закрывает приложение.

### 5.2 Шаблон переименования (v1.1)
Пользователь жмёт «Экспорт → переименовать»:
- выбирает целевую папку,
- выбирает или собирает шаблон имени с переменными:
  - `{group}` — имя группы,
  - `{index:003}` — порядковый номер с фиксированным числом разрядов,
  - `{date}` — дата (съёмки из EXIF или экспорта, на выбор),
  - `{original}` — исходное имя файла без расширения,
- предпросмотр итоговых имён,
- запуск копирования (исходники остаются нетронутыми).

Примеры результирующих имён:
- `wedding_001_2026-05-24.jpg`
- `prep_001_DSC_4521.jpg`

## 6. Технические требования

### 6.1 Платформа
- macOS — **только Apple Silicon** (arm64). Intel-сборка не выпускается.
- Windows 10 / 11 (x64).

### 6.2 Технологический стек
- **Frontend:** React 18 + TypeScript, Vite, Tailwind CSS
- **State management:** Zustand
- **Drag & Drop:** `dnd-kit`
- **Desktop runtime:** Tauri (Rust + WebView)
- **Backend (Rust):** обработка файлов, генерация миниатюр, извлечение цветов, экспорт
- **Image processing (Rust crates):** `image`, `kmeans_colors` или `color-thief-rs`, `rayon` для параллелизма
- **Хранилище:** локальные JSON-файлы проекта (`.bgrid`) + кэш миниатюр в папке проекта/AppData
- **Сборка и распространение:** GitHub Releases (DMG для Mac, MSI для Windows), CI через GitHub Actions
- **Лицензия:** MIT

### 6.3 Авторизация и сеть
- Отсутствуют. Приложение полностью офлайн.

## 7. UX / Дизайн (общие принципы)

- Минималистичный современный интерфейс, тёмная тема по умолчанию (важно для оценки фото).
- Главные экраны: **Library** (вся библиотека проекта), **Grid** (рабочая сетка), **Export** (настройки экспорта).
- Реактивный отклик: миниатюры подгружаются по мере готовности, drag&drop без лагов даже на 500 фото (виртуализация списка).
- **Индикатор импорта/обработки:** общий процент + численный счётчик в формате `25 / 600`.
- **Иконка и брендинг:** генерируем с нуля (нет существующих наработок). Стиль — минималистичный, отсылка к сетке/композиции.
- **Кнопка «Поддержать»** доступна сразу в нескольких местах (шапка окна, меню «About», настройки), открывает выбор платформы (Buy Me a Coffee / Boosty / GitHub Sponsors) во внешнем браузере. При первом запуске — одноразовое вежливое модальное окно (см. §4.3).

## 8. Метрики успеха

- Открытие проекта на 300 фото < 5 сек (после первой обработки).
- Drag&drop без видимых лагов на 500 фото.
- Экспорт сетки 4×N (N=20–30) < 15 сек.
- ≥ 50 GitHub Stars в первые 3 месяца после релиза v1.0 (для open-source kudos).

## 9. Нефункциональные требования

- **Локальность.** Никаких сетевых запросов с пользовательскими данными.
- **Безопасность исходников.** Оригинальные файлы никогда не изменяются и не удаляются приложением.
- **Производительность.** Параллельная обработка фото (Rust + Rayon).
- **Кроссплатформенность.** Один кодбейз, идентичный UX на Mac и Windows.
- **Открытость.** Весь код в публичном репозитории, MIT-лицензия, прозрачные релизы.

## 10. Ограничения и риски

| Риск | Митигация |
|---|---|
| Большой объём фото может тормозить UI | Виртуализация списков, фоновая обработка в Rust |
| RAW и редкие форматы | Вне MVP, добавляем позже |
| Скорость экспорта больших коллажей | Использовать `image` crate с тайловой записью |
| Tauri ещё молодая платформа | Активное сообщество, стабильная v2 |
